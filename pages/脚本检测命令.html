✅ 脚本检测命令

📖 什么是脚本检测命令？

脚本检测命令用于在 #IF 条件判断中检测各种游戏状态、玩家属性、物品数量等。

检测命令通常在NPC对话或触发脚本的 #IF 部分使用，满足条件时执行 #ACT 中的动作。

💡 示例：武斗场进入条件检测

这个示例展示了如何组合使用多个检测命令来实现复杂的条件判断。

; 检测是否为周一、周三、周五（使用#OR实现"或"逻辑）
[@进入武斗场]
#OR
检测变量 <$星期> == 1
检测变量 <$星期> == 3
检测变量 <$星期> == 5
#ACT
#SAY
每周一、三、五的21点，等级达到25级的勇士可进入武斗场。
#ELSESAY
还未到开放时间，请稍安勿躁。每周一、三、五的晚上21点开放。

; 检测等级条件
[@确认进入]
#IF
检测角色等级 < 25
#ACT
BREAK
#SAY
你的等级未满[25]级，暂时无法进入。

; 检测金币数量
#IF
检测金币数量 < 10000
#ACT
BREAK
#SAY
身上不足[10000]金币，不足以支付报名费用！

; 检测时间段（多个条件组合，默认为"且"关系）
#IF
检测变量 <$小时> == 21
检测变量 <$分钟> < 30
#ACT
调整角色金币 - 10000
传送指定地图 183 0 0 3
#ELSESAY
还未到开放时间，每周一、三、五的晚上21点到21点30分开放。
💡 使用说明与示例
基本用法：
• 检测命令通常在 #IF 部分使用
• 多个检测命令可以组合使用，默认为"且"关系
• 使用 #OR 可以实现"或"关系
• 检测命令的具体参数格式请参考对应命令的详细文档
📌 关键要点：
1. #OR 逻辑：使用 #OR 可以实现"或"关系，满足任一条件即可
2. 多条件组合：在 #IF 中连续写多个检测命令，默认为"且"关系
3. BREAK 命令：用于中断执行，不满足条件时终止后续判断
4. #ELSESAY：条件不满足时显示的内容
